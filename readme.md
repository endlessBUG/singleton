1.饿汉模式 饿汉模式线程安全 体现在其再初始化的时候就已经赋值 但是再未使用的时候浪费内存
2.懒汉模式 再初次调用的时候赋值 懒加载 线程不安全 体现在 Object != null 上 可能有多个线程同时判断成功 导致实例化的时候retrun 不同的对象 
正常懒汉模式(syc) 
double check模式 之所以效率较上着较高是因为 锁粒度变小了 第一次!=nul的时候只能进来部分少数线程,再将这少数线程锁了一次判断 即可理解为锁的线程少了 
 静态内部类的形式 内部类再初次调用的时候初始化 符合懒加载模式 属性final 因而安全
3.登记式单例 采用map key,value的形式 用 syc 保证key不重复 因而安全 可以采用doublecheck 减小锁力度
4.枚举单例 最安全的单例模式  反编译.class文件采用饿汉模式  jdk底层不允许实例化 反序列化的时候单独对枚举做处理 因而安全
